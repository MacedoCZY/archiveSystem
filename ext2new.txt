a princípio eu olho no inicio 1024 ou 0x400, para ver onde esta o Super Block SB
se ele estiver no zero junto com o Boot dir está tudo tranquilo e se encaixa neste exemplo
se estiver em 1 e, consequentemente, o GD estará na posição 2, se encaixa no exeplo anterior

inicia deslocando 1024 do boot record, ou ox400

block size = 00|00 00|01   at >> (24, 27) 4 bytes = 1 isto é o número de << que vou dar em 1024, portanto = 2048

blocks por grub = 00|40 00|00 at >> (32, 35) 4 bytes

num indoes per group = 00|00 14|00 at >> (40, 43) 4 bytes

total inodes = 00|00| 28|00 at >> (0, 3) 4 bytes

total blocks = 00|00 50|00 at >> (4, 7) 4 bytes

sabemos que cada block tem 2048 bytes, e que o GD(Grub Descriptor)
está na posição 1 o SB(Super Block) está em 0 e ocupa 1 block
então o GD incia em 2048, logo vamos na posição 0x800 = 2048

no GD temos 0x15 DB(Data Block) 0x16IB(Inode Block) 
0x17IT(Inode Table)

Para encontrar o inicio da Inode Table pegamos o
0x17 x 0x800 = B8|00

O Diretorio raiz está sempre no Inode(2) por convenção
do ext2, portanto quando chegamos no inicio do Inode table
pulamos +128 para analisarmos apartir do segundo inode

agora aqui estamos no inicio do segundo inode
DB0 = 00|00 01|57 at >> (40, 43)
DB1 = 00|00 00|00 at >> (44, 47) ... fim da TI

Para podermos analisar os dados agora, vamos multiplicar
o DB0 pelo Block Size
00|00 01|57 x 00|00 08|00 = 0xab800

Agora vou ir no 0xab800 e processar os dados la de dentro
